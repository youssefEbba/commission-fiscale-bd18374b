# Internationalisation (FR / AR)

Setup i18next + react-i18next, FR par défaut, AR avec RTL natif.

## Architecture

```
src/i18n/
  index.ts              # Init i18next, detector, namespaces
  bootstrap.tsx         # <I18nBootstrap/> : sync <html lang/dir>, persistance
  format.ts             # formatDate, formatDateTime, formatAmount, formatNumber
  enums.ts              # Helpers de traduction des enums métier
  LanguageSwitcher.tsx  # Sélecteur de langue (header)
  locales/
    fr/{common,nav,auth,certificats,utilisations,...}.json
    ar/{common,nav,auth,certificats,utilisations,...}.json
```

Namespaces : `common`, `nav`, `auth`, `conventions`, `marches`, `demandes`, `certificats`, `utilisations`, `transferts`, `cloture`, `modifications`, `ged`, `users`, `roles`, `audit`, `referentiel`, `errors`, `enums`.

## Utilisation

```tsx
import { useTranslation } from "react-i18next";

function MyPage() {
  const { t } = useTranslation("certificats");
  return <h1>{t("list.title")}</h1>;
}

// Cross-namespace : préfixer
const { t } = useTranslation(["certificats", "common"]);
t("common:actions.save");
```

### Interpolation

```tsx
t("session.impersonating", { label: "ACME" }); // → « Vous agissez en tant que ACME »
```

### Pluriels (i18next + Intl.PluralRules)

Définir des clés `key_one`, `key_other` (FR) et `key_zero`, `key_one`, `key_two`, `key_few`, `key_many`, `key_other` (AR).

```json
"items_count_one": "{{count}} élément",
"items_count_other": "{{count}} éléments"
```

```tsx
t("items_count", { count: n });
```

## Enums

Les valeurs renvoyées par l'API (`OUVERT`, `DGTCP`, …) restent dans la donnée. Le front les traduit via les helpers de `src/i18n/enums.ts` :

```tsx
import { tStatutCertificat, tRole } from "@/i18n/enums";

<Badge>{tStatutCertificat(cert.statut)}</Badge>
<span>{tRole(user.role)}</span>
```

Ajouter un nouveau mapping : créer la clé dans `enums.json` (`fr` + `ar`), puis exposer un helper dans `enums.ts`.

## Erreurs API

`showApiError(err)` cherche d'abord `errors:<code>` puis retombe sur le message brut du backend.

```ts
import { showApiError } from "@/lib/feedback";
showApiError(err); // titre i18n par défaut
```

## Format dates / montants

```ts
import { formatDate, formatDateTime, formatAmount, formatNumber } from "@/i18n/format";

formatDate("2026-05-18");            // « 18/05/2026 » (fr) — « 18/05/2026 » (ar, chiffres latins)
formatAmount(1500000);               // « 1 500 000 MRU »
formatAmount(1500000, { currency: "USD" });
```

Pour forcer les chiffres arabes : `formatAmount(n, { numberingSystem: "arab" })`.

## RTL

- `<I18nBootstrap/>` met à jour `<html lang|dir>` automatiquement.
- Tailwind v3 fournit les variants `rtl:` et `ltr:` natifs (à utiliser pour les inversions de mise en page).
- **Convention** : préférer les utilitaires logiques (`ms-`/`me-`, `ps-`/`pe-`, `start-`/`end-`, `text-start`/`text-end`) plutôt que `ml-`/`mr-`, etc.
- Icônes directionnelles (chevrons, flèches « → ») : utiliser `rtl:rotate-180` ou un fallback texte.
- Police arabe : `Noto Sans Arabic` chargée via `@fontsource`, appliquée par `html[lang="ar"] body { font-family: ... }`.

## Ajouter une clé

1. Ajouter la clé dans `src/i18n/locales/fr/<ns>.json`.
2. Ajouter la traduction dans `src/i18n/locales/ar/<ns>.json` (même chemin).
3. Utiliser `t("<ns>:<key>")`.
4. Vérifier : `node scripts/i18n-check.mjs`.

## Ajouter une langue

1. Créer `src/i18n/locales/<lang>/` avec les mêmes fichiers que `fr/`.
2. Ajouter le code dans `SUPPORTED_LANGS` (`src/i18n/index.ts`) et importer les JSON.
3. Ajouter l'option dans `LanguageSwitcher.tsx`.
4. Si RTL : adapter `bootstrap.tsx` (déjà géré pour `ar`).

## Outillage

- `node scripts/i18n-check.mjs` : compare FR vs AR, sortie code = 1 si AR incomplet.
- En dev, les clés manquantes sont loguées dans la console (`[i18n] Missing key`).

## Convention de nommage

- Clés en anglais, `snake_case` ou `kebab-case` selon contexte.
- Hiérarchie : `<page>.<section>.<element>` (ex. `certificats.list.title`).
- Actions communes : `common.actions.*`.
- Statuts : `enums.<enum_name>.<VALEUR_BACKEND>`.
- Erreurs API : `errors.<code_minuscule>` (mappé sur `ApiError.code`).
